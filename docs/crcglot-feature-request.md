# Feature Request: Generation Metadata in Module Docstrings

## Motivation
When crcglot-generated code is embedded in downstream projects, developers need to know:
- Which crcglot version generated this file
- Which exact flags and algorithm were used
- How to reproduce the generation

This information should be discoverable via standard documentation tools (Python `help()`, IDE inspection, etc.) and consistent across all target languages.

## Proposal
Add an optional `--prov="<command>"` flag that embeds generation metadata in the module/file docstring of every generated file.

The metadata section should appear early in the docstring, formatted consistently across all languages:

### Python example
```python
"""crc16-xmodem -- XMODEM, ZMODEM, ACORN, LTE

Parameters: width=16, poly=0x1021, init=0x0000, refin=false, refout=false, xorout=0x0000.
check: crc16_xmodem(data) over "123456789" == 0x31C3

Generated by crcglot:
    tool: crcglot
    tool_version: 0.20.0
    algorithm: crc16-xmodem
    target: python
    variant: small
    comment: google
    symbol: crc16_xmodem
    command: uv run crcglot python crc16-xmodem --small --comment google

Example:
    ...
"""
```

### C example
```c
/**
 * crc16-xmodem -- XMODEM, ZMODEM, ACORN, LTE
 *
 * Parameters: width=16, poly=0x1021, init=0x0000, refin=false, refout=false, xorout=0x0000.
 * check: crc16_xmodem(data) over "123456789" == 0x31C3
 *
 * Generated by crcglot:
 *     tool: crcglot
 *     tool_version: 0.20.0
 *     algorithm: crc16-xmodem
 *     target: c
 *     variant: small
 *     comment: doxygen
 *     symbol: crc16_xmodem
 *     command: crcglot c crc16-xmodem --small --comment doxygen
 *
 * Example:
 *     ...
 */
```

## Implementation Approach

### Create an internal helper function
Generate the canonical metadata block as plain text:

```python
def _generate_metadata_block(
    tool_version: str,
    algorithm: str,
    target: str,
    variant: str,
    comment_style: str,
    symbol: str,
    command: str,
) -> str:
    """Generate the standardized metadata block (without language-specific comment syntax).
    
    Returns multi-line text with consistent formatting and key ordering.
    Caller is responsible for wrapping in language-appropriate comment chars/indentation.
    """
```

This helper returns plain text like:
```
Generated by crcglot:
    tool: crcglot
    tool_version: 0.20.0
    algorithm: crc16-xmodem
    target: python
    variant: small
    comment: google
    symbol: crc16_xmodem
    command: uv run crcglot python crc16-xmodem --small --comment google
```

### Language generators
Each language generator calls the helper and wraps the result with language-specific comment syntax and indentation. For example:

- Python: indent with 0 spaces (inside triple-quote docstring)
- C: indent with 1 space per ` * ` prefix
- TypeScript: indent inside `/** ... */`
- Rust: indent inside `/// ...` or `//! ...`

## Key Points
- **No timestamps or user info** unless explicitly requested by a future `--created-by` flag.
- **Deterministic field order** for easy diffing and parsing.
- **Language-specific formatting only:** The helper provides raw text; language generators handle comment syntax and indentation.
- **Placement:** Early in the docstring, after the algorithm title/description, before the main examples.
- **Opt-in via `--prov`:** If flag is not provided, no metadata section is added (backward compatible).

## Example Usage
```bash
crcglot python crc16-xmodem --small --comment google --prov="uv run crcglot python crc16-xmodem --small --comment google"
crcglot c crc16-xmodem --small --comment doxygen --prov="crcglot c crc16-xmodem --small --comment doxygen"
```
